## Overview

A personal project building a 2D game engine from the ground up using SDL2 and C. This is a long-held dream project that lets me explore low-level programming, graphics rendering, and game architecture outside of my day-to-day web development work.

## Why a Game Engine

After spending most of my career in high-level languages and web frameworks, I wanted to go back to basics. There's something deeply satisfying about working close to the metal: managing memory manually, thinking about render loops and frame timing, and building abstractions from first principles.

Game engines are one of the most interesting software architecture challenges. They require real-time performance, clean separation of concerns (rendering, physics, input, audio), and careful resource management. It's the kind of problem that makes you a better engineer in every other domain.

## Current State

The engine currently handles basic sprite rendering, input handling, and a fixed-timestep game loop. I'm working on entity-component systems and basic collision detection. It's very much a work in progress, the kind of project that might never be "done," and that's the point.

## What I'm Learning

Working in C after years of TypeScript and React is a humbling and refreshing experience. It forces clarity of thought in a way that garbage-collected languages don't. Every allocation is intentional. Every pointer tells you something about ownership. It's making me a more thoughtful programmer across the board.
